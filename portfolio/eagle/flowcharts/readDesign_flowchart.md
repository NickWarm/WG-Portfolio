# readDesign 執行流程

```
/readDesign 執行流程
│
├─ 1. 【參數解析】
│   ├─ 無參數 → 顯示設計規則
│   └─ 有參數 → 進入查詢模式
│
├─ 2. 【查詢模式】
│   ├─ 讀取定義檔 commands/{skill}.md
│   ├─ 從 frontmatter 提取 design-doc 路徑（記住但不讀）
│   ├─ 讀取流程圖 flowcharts/{skill}_flowchart.md（如存在）
│   └─ 顯示定義檔結構 + 流程圖內容
│
├─ 3. 【輸出】
│   ├─ 無參數 → 顯示完整設計規則
│   └─ 有參數 → 定義檔 + 流程圖摘要 + 設計稿路徑提示
│
└─ 4. 【後續自動行為】
    └─ 用戶表達修改意圖 → AI 自動讀取設計稿 → 進入修改流程
        └─ 大檔案讀取規則（設計稿 > 2000 行時）
            ├─ wc -l 確認行數
            ├─ ≤ 2000 行 → 直接 Read
            └─ > 2000 行 → 分段 Read（offset + limit=2000）
```
