# read-data-flow 執行流程

```
/read-data-flow {模組名稱或絕對路徑} [模組名稱或絕對路徑2 ...] 執行流程
│
├─ 1. 【參數解析】
│   ├─ 無參數 → 列出映射表所有可用模組，提示用戶選擇
│   └─ 有參數 → 逐一判斷每個參數
│       ├─ 以 / 開頭 → 絕對路徑模式，直接使用
│       └─ 否則 → 模組名稱模式
│           └─ 查映射表
│               ├─ 找到 → 使用對應路徑
│               └─ 找不到 → 報錯，列出所有可用模組名稱
│
├─ 2. 【檔案驗證】
│   ├─ 逐一檢查解析後的每個路徑是否存在
│   ├─ 有不存在的 → 列出映射表所有可用模組
│   └─ 全部通過 → 繼續
│
├─ 3. 【讀取檔案】
│   ├─ 單檔 → 直接分段讀取（每段 600 行，使用 offset/limit）
│   └─ 多檔 → 並行 Task agent 各讀一個檔案
│       └─ 每個 agent 回傳：基本資訊 + 問題清單原文
│
├─ 4. 【問題摘要】
│   ├─ 4.1 提取基本資訊
│   │   ├─ Module 名稱（從 ## 基本資訊 區塊）
│   │   ├─ API 目錄（adminApi / publicApi）
│   │   └─ 對應前端專案（dashboard-nuxt / frontend-nuxt）
│   ├─ 4.2 找到「## 問題清單」區塊
│   │   ├─ ### ❌ 開頭 → 嚴重問題
│   │   ├─ ### ⚠️ 開頭 → 注意事項
│   │   └─ ### ℹ️ 開頭 → 設計備註
│   ├─ 4.3 解析每個子區塊的表格行
│   │   └─ 提取：#、嚴重度、類型、問題描述、位置/影響
│   └─ 4.4 彙整跨檔案統計（多檔時）
│       └─ 按嚴重度 × 模組 統計數量
│
└─ 5. 【輸出結果】
    ├─ 檔案基本資訊表
    ├─ 問題統計表（嚴重度 × 模組 + 總計）
    ├─ ❌ 嚴重問題詳細列表（含 Module、類型、問題描述、影響）
    ├─ ⚠️ 注意事項詳細列表
    └─ ℹ️ 設計備註（簡述）
```
