# /showFlow 執行流程

```
/showFlow 執行流程
│
├─ 1. 【解析參數】
│   ├─ $1 = u → 進入 Update 模式（Step 2）
│   └─ 其他 → 進入顯示模式（Step 3）
│
├─ 2. 【Update 模式：建立進度表】（/showFlow u {type}）
│   ├─ 2.1 定位 Proposal
│   │   ├─ 從對話上下文找到當前 proposal 路徑
│   │   └─ 找不到 → 提示用戶指定
│   ├─ 2.2 檢查置頂有無進度表
│   │   ├─ 有「## 工作流程進度」→ 回報已存在，結束
│   │   └─ 沒有 → 繼續
│   ├─ 2.3 建立進度表骨架（標題後、正文前）
│   │   ├─ $2 = debug → Bug Fix 流程表（全部 ⏸️）
│   │   └─ $2 = dpf → Data Flow 驅動流程表（全部 ⏸️）
│   └─ 2.4 輸出確認
│       └─ ✅ 進度表已建立於 {proposal 檔名}
│
└─ 3. 【顯示模式：印出流程圖】（原有邏輯）
    ├─ 無參數 → 印出三條流程總覽
    ├─ $1 = api → 印出 API Data Flow 建立詳細流程
    ├─ $1 = df → 印出 Data Flow 驅動修復詳細流程
    ├─ $1 = bug → 印出 Bug Fix 詳細流程
    ├─ 從對話上下文偵測當前進度（best effort）
    │   ├─ 有進度 → ✅ 已完成 + 👉 當前步驟
    │   └─ 無進度 → 只印流程圖
    └─ 提示下一步指令
```
