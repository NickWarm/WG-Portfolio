# /epl Skill 設計稿

> **Skill 名稱**：epl (explain proposal)
> **建立日期**：2025-01-29
> **狀態**：已實作

## 問題描述

在討論設計方案時，AI 容易直接進入細節，缺乏全貌整理。用戶需要一個快速指令，讓 AI 先整理問題、再用流程圖說明解法架構。

## 執行流程

```
/epl 執行流程
│
├─ 1. 【分析上下文】
│   └─ 從對話中提取當前討論的核心問題
│
├─ 2. 【問題摘要】
│   ├─ 一句話描述問題本質
│   └─ 列出關鍵限制或條件
│
└─ 3. 【解法流程圖】
    └─ 用有序樹狀格式說明解決方案
```

## 參數說明

無參數，直接根據對話上下文執行。

## 輸出格式

```markdown
## 🎯 問題摘要

**核心問題**：[一句話描述]

**關鍵條件**：
- [條件 1]
- [條件 2]

## 📊 解法流程圖

[有序樹狀流程圖]
```

## 實作細節

- 流程圖必須使用有序樹狀格式（禁止箭頭式）
- 問題摘要應簡潔，不超過 3 個關鍵條件
- 流程圖層級建議 2-4 層，避免過度複雜
