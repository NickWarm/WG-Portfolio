# 提案範本

複製整個 `_template/` 資料夾，改名為公司代號（例如 `acme/`），再填入內容。

每份提案兩個檔案：

- `analysis.md` — 我對該公司／產業公開資訊的拆解，找出可被 agent 化的痛點
- `design.md` — 針對拆解結果，我會怎麼設計 agent / skills / 流程

## 誠實標示

提案開頭必須註明：

> **本提案基於公開資訊構想，未在該公司實際驗證。**
> 與 `experience/` 中已實戰的系統不同，這裡呈現的是「我會如何思考與設計」的能力，
> 而非「這個解法已經被證實有效」。

這不是免責聲明，是給讀者正確的閱讀框架——也是專業誠實的展現。
