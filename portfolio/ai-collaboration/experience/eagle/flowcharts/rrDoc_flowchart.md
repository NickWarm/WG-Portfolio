# rrDoc 執行流程

**rrDoc** = **R**ead **R**eview Result + Update **Doc**

```
/rrDoc 執行流程
│
├─ 1. 【檢查前置條件】
│   ├─ 有 /reviewDoc 結果 → 繼續
│   └─ 沒有 → 提示「請先執行 /reviewDoc」並中止
│
├─ 2. 【取得上次檢核結果】
│   ├─ 從對話上下文找到最近一次 /reviewDoc 的結果
│   └─ 提取「❌ 發現的問題」區塊中的修改建議
│
├─ 3. 【更新文件】
│   ├─ 逐項處理修改建議
│   │   ├─ 可直接修正 → 執行修正
│   │   └─ 需用戶確認 → 標註「⚠️ 待確認」
│   └─ 記錄每項修改內容
│
├─ 4. 【再次檢核】
│   └─ 嵌套調用 /reviewDoc again
│
└─ 5. 【輸出結果】
    ├─ 本次修正摘要
    ├─ 已更新文件清單
    ├─ /reviewDoc 檢核結果
    └─ 後續操作提示
```

## 修正項目分類

| 類型 | 處理方式 |
|------|----------|
| DTO description 補充 | ✅ 直接修正 |
| DTO example 補充 | ✅ 直接修正 |
| 欄位一致性（create/update 同步） | ✅ 直接修正 |
| 文件結構調整（區塊順序） | ✅ 直接修正 |
| 欄位影響矩陣補充 | ✅ 直接修正 |
| Service 層 select 補充 | ✅ 直接修正 |
| 業務邏輯問題 | ⚠️ 標註待確認 |
| 架構設計問題 | ⚠️ 標註待確認 |
