# implement 執行流程

```
/implement 執行流程
│
├─ 0. 【建立進度追蹤】（強制）
│   └─ TaskCreate 建立 8 個步驟
│
├─ 1. 【判斷提案文件】
│   └─ 從對話上下文判斷要實作的提案
│
├─ 2. 【讀取提案文件】
│   └─ 理解修改內容
│
├─ 3. 【分類修改項目】
│   ├─ 後端修改（backend-nestjs）→ 加入執行清單
│   └─ 前端修改（dashboard-nuxt / frontend-nuxt）→ 🚫 不執行，標註「待處理」
│
├─ 4. 【分段實作後端程式碼】
│   ├─ 4a. 分析 proposal 結構 → 拆分 sub-task
│   ├─ 4b. 逐段實作（讀取 → 修改 → 完成）
│   └─ ⚠️ 禁止長時間分析不動工
│
├─ 5. 【執行 Build 驗證】
│   ├─ yarn build
│   ├─ 成功 → 繼續
│   └─ 失敗 → 修正後重新 build
│
├─ 6. 【雙重檢核】
│   ├─ 第一次：重新讀取提案，逐項對照
│   └─ 第二次：確認沒有做提案範圍外的修改
│
├─ 7. 【更新提案文件】
│   ├─ 已完成項目加 ✅ 和完成日期
│   ├─ 更新狀態（缺少 → 已新增）
│   └─ 新增「實作進度」區塊
│
├─ 8. 【結束前檢查】（強制）
│   ├─ TaskList 確認所有 task 狀態
│   ├─ 全部 completed → 輸出結果
│   └─ 有未完成 → 先補完成再輸出
│
└─ 9. 【輸出結果 + 後續提醒】
    ├─ ✅ 實作完成項目（後端）
    ├─ 🚫 前端修改（待處理）
    ├─ ✅ 雙重檢核結果
    ├─ ✅ Build 結果
    └─ 提醒：/check-result → /gcommit-push
│
└─ 10. 【更新進度表】（@update-progress.md）
    └─ 將 /implement 標記為 ✅
```
